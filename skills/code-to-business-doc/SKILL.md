# 代码业务说明文档生成器

将代码实现转化为面向测试和运维人员的业务说明文档。

## 工作流程

### 1. 分析代码

阅读用户指定的代码文件或模块，识别：
- 核心业务逻辑和算法
- 输入/输出数据结构
- 配置项和参数
- API 接口定义
- 关键边界条件

### 2. 生成业务说明文档

使用以下模板结构输出文档：

```markdown
# [功能名称] 业务说明

## 核心价值
> 简要说明这个功能给用户/业务带来的价值，比如：实现了什么能力、提升了什么 KPI、解决了什么痛点。

## 功能概述
简要描述功能的目的和作用。

## 核心业务逻辑
用流程或步骤描述主要逻辑，避免代码细节，使用业务语言。

## 输入/输出说明
| 类型 | 名称 | 说明 | 示例值 |
|------|------|------|--------|
| 输入 | xxx | xxx | xxx |
| 输出 | xxx | xxx | xxx |

## 配置项说明
| 参数名 | 说明 | 默认值 | 取值范围 |
|--------|------|--------|----------|
| xxx | xxx | xxx | xxx |

## 接口说明（如适用）
- **接口路径**: `POST /api/xxx`
- **请求参数**: ...
- **响应示例**: ...

## 使用场景示例
描述 1-3 个典型的使用场景，帮助测试人员设计用例。

## 注意事项
- 边界条件和限制
- 常见问题和解决方案
- 依赖关系
```

## 写作原则

1. **业务视角**: 用业务语言描述，避免代码术语
2. **具体实用**: 提供具体的参数值、示例，而非抽象描述
3. **面向受众**: 测试人员关心"怎么测"，运维人员关心"怎么配置/排错"
4. **简洁清晰**: 每个章节内容精炼，不堆砌

## 输出格式

使用中文输出文档，格式为 Markdown。
