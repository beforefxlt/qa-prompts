# 自动化测试交付流水线 (Automated Test Lifecycle)

这是一个标准的测试交付流水线 (SOP)。当用户提供新的需求或更新的特性时，按照以下步骤严格执行：

1. **获取信息与需求审计 (只做审计)**：
   - 首先收集用户提供的需求描述内容。
   - 调用 `requirement-reviewer` Skill，重点审查逻辑真空和 P3/P5 边缘情况。
   - **【落地建议 4：多存中间文件】**：将生成的审计表格保存为临时文件 `/tmp/requirement_audit.md`。
   - 向用户汇报并展示该文件，询问：“您需要补充相关异常情况的处理逻辑，还是直接基于当前设定生成测试策略？”
   - **【验证标准】**：等待用户明确回复“继续”或解决掉高优风险点后再继续。

2. **输出全景测试策略 (只做架构设计)**：
   - 读取刚才的审计结果，调用 `test-strategy-planner` Skill。
   - 严格按照 P1 到 P5 输出覆盖测试全场景的矩阵策略大纲。
   - **【落地建议 4：多存中间文件】**：将策略大纲保存到临时文件 `/tmp/test_strategy_draft.md`。
   - 将策略大纲展示给用户，并说明：“这是为您设计的全局测试策略网，如果方向无误，我将开始为您大规模生成详细用例。”
   - **【验证标准】**：等待用户确认（Approval）矩阵内容后，再授权执行下一步。

3. **批量生成标准测试用例 (只做执行)**：
   - **【落地建议 2：Skill 原子化】**：严格读取 `/tmp/test_strategy_draft.md` 作为唯一输入上下文，调用 `test-case-factory` Skill。
   - 严格参考项目约定的用例模板，生成可以直接导入系统（如 Jira/TestLink）的 TC-001 标准测试用例（包括明确的隔离配置前提和报错验证要求）。
   - 将生成的用例表格直接写入到正式文档 `docs/test_specification.md` 分栏中，并要求用户前往复核。
