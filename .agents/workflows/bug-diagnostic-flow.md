# AI 缺陷初诊与排障隔离流水线 (Bug Diagnostic Flow)

> **触发条件**：当测试、实施或运维只提供了一个简短的缺陷现象（如“通讯断了”“程序崩了”“数值不对”），你需要启动本流程，将模糊现象转化为可追踪、可复核的缺陷隔离结论与 `BUG-001` 草稿。
> **核心原则**：本工作流严格执行“物理状态存证”与“上下文净化”。每一阶段都应将阶段结论落到 `/tmp`，下一阶段只读取上一步的物理结果继续分析，降低长对话带来的脑补风险。
> **资产边界**：`BUG-001` 缺陷分析单属于单次诊断产物，默认**不得直接落入 Git 仓库**。正式问题单应优先停留在 `/tmp`、外部 Issue 系统或用户明确指定的非仓库交付路径。

## Step 1: 症状初诊与证据收集 (Information Gathering)
此阶段的目的是做“问诊”，绝不允许直接写 Bug 报告。你需要向用户索要能证明问题的核心证据。

1. **审视初始输入**：阅读用户的缺陷描述。
2. **定位缺失项**：判断工控场景中常见的“控制盲区”（如：发生问题前系统的运行状态是稳态还是在做切换？硬件的具体版本号是什么？发生问题时的网络状态如何？是否有相关微服务的错误日志？）。
3. **交互追问**：向用户列出 3-4 个关键追问问题，**要求提供明确的日志路径或截图**。
4. **状态落盘**：当用户答复后，**不要进行下一步推演**。立即将“原始现象 + 用户补充的所有答案与日志”合并整理，**写入本地的临时物理文件：`/tmp/bug_investigation_raw.md`**。

> ⚠️ **用户确认点**：请回复：“初始病情已固化至 `/tmp/bug_investigation_raw.md`。请回复 **‘开始推演’** 以进行根因隔离分析。”

## Step 2: 根因推演与多维隔离排查 (Root Cause Brainstorming)
在此阶段，AI 扮演高级测试架构专家，对现场证据进行逻辑闭环的推理定性。

1. **上下文净化 (Context Sanitization)**：从此刻起，不再依赖上游对话原话，只读取上一步生成的 `/tmp/bug_investigation_raw.md` 作为唯一事实依据。
2. **D1-D5 维度病理分析**：
   - 检查日志是抛出了底层 Exception 还是业务代码里的拦截异常。
   - 推演问题是由于环境（D3：无权限、断网、组件缺失）引起的，还是边界条件（D4：数值越限、并发竞争、时序边界）触发的。
3. **输出定论**：将推演的结论（可能是初步推论），包括**发生条件、疑似代码模块/组件、对系统的严重影响（宕机/降级）**，严格写入物理文件：**`/tmp/bug_root_cause_analysis.md`**。

> ⚠️ **用户确认点**：向用户展示您的初诊隔离分析结论（摘要），询问：“是否同意按照此方向进行定级与正式建单？请回复 **‘建单’**。”

## Step 2.5: 严重程度定级 (Severity Triage)
在进入正式建单前，依据 `bug-triage-rules` skill 对缺陷进行严重程度定级。

1. **读取根因结论**：从 `/tmp/bug_root_cause_analysis.md` 中提取事实。
2. **执行四步决策树**：调用 `bug-triage-rules` skill，依次评估安全性影响、数据完整性、业务阻断度、功能正确性四个因子，输出 P0/P1/P2 等级。
3. **定级结果落盘**：将定级结论（等级、命中因子、判定依据）追加写入 `/tmp/bug_root_cause_analysis.md` 末尾。
4. **用户确认**：向用户展示定级结论，询问是否同意。若用户对等级有异议，以用户判定为准。

## Step 3: 标准缺陷报告草稿生成 (Formalize BUG-001 Draft)
此阶段为标准化建单操作。此时事实已经基本清楚，但仍需明确区分“正式仓库资产”与“单次诊断草稿”。

1. **终极上下文隔离**：忽略前面两轮的聊天原文，仅依靠读取 `/tmp/bug_root_cause_analysis.md` 中的结论。
2. **调用标准化技能**：优先调用 `issue-reporter`，按 `templates/defect-report-template.md` 生成 `BUG-001` 草稿。
3. **草稿落盘**：将最终生成的正规 Markdown 草稿写入 `/tmp/bug_report_draft.md`。
4. **交付边界确认**：向用户说明该草稿默认属于单次诊断产物，不直接进入 Git 仓库；若用户要同步到 Issue 系统或导出到指定目录，再单独确认目标位置。

> 🏁 **完成提示**：告诉用户：“缺陷隔离结论与 `BUG-001` 草稿已生成并固化至 `/tmp/bug_report_draft.md`。如需继续，我可以按该草稿协助您同步到外部 Issue 系统或指定的非仓库交付路径。”
