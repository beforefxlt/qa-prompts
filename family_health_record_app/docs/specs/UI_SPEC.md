# 家庭检查单管理应用 UI 规格

## 1. 设计原则

- 移动端优先
- 首屏直达核心信息，不让用户先学系统
- 图表强调参考区间、报警红区和变化率
- OCR 审核页强调编辑效率与可追溯性

## 2. 信息架构

- 首页 (家庭成员概览)
- 成员档案页
- 上传页
- OCR 审核页
- 趋势页
- 单次检查详情页

## 3. 页面定义

### 3.1 首页

展示内容：
- 成员卡片（支持快速切换）
- 最近一次检查时间
- 待审核检查单数
- 重点指标预警汇总

### 3.2 成员档案页

展示内容：
- 基础资料（含年龄、性别、档案类型）
- 检查时间轴
- 指标快捷入口
- 最近 3 个月趋势摘要

### 3.3 上传页

展示内容：
- 上传按钮
- 文件类型说明
- 脱敏处理中状态
- OCR 处理中状态

### 3.4 OCR 审核页

展示内容：
- 左侧原始脱敏图预览
- 右侧结构化字段表单
- 置信度提示
- 单位错误或规则冲突提示
- 审核通过 / 退回 / 保存草稿操作

### 3.5 趋势页

展示内容：
- 时间范围切换
- 指标切换
- 图表与参考区间
- 报警说明
- 上次检查对比

## 4. 图表策略

### 4.1 生长发育

- 身高折线图
- 体重折线图
- 标准百分位参考带
- 年增长速度摘要

### 4.2 视力与眼轴

- 眼轴趋势图
- 视力或屈光度趋势图
- 左右眼并列展示
- 参考区间与异常提示（基于成员年龄动态适配）

### 4.3 血糖与血脂

- 折线图叠加正常区间
- 超阈值数据点高亮红色
- 指标越界说明卡片

## 5. 状态设计

- 空状态：无检查单、无趋势数据、无待审核项
- 处理中：上传中、脱敏中、OCR 中、校验中
- 异常状态：识别失败、规则冲突、待人工审核
- 成功状态：审核完成并入库

## 6. 前端实现要求

- 前端可先基于 Mock Data 开发
- 页面组件必须与 API Contract 解耦
- 图表配置集中管理，避免散落在页面代码中
