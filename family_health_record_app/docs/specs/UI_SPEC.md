# 家庭检查单管理应用 UI 规格

## 1. 设计原则

- 移动端优先
- 首屏直达核心信息，不让用户先学系统
- 图表强调参考区间、报警红区和变化率
- OCR 审核页强调编辑效率与可追溯性
- 内网免登录，打开即用

## 2. 信息架构

- 首页 (家庭成员概览 / 空状态引导)
- 成员档案创建/编辑页
- 成员 Dashboard 页
- 上传页（弹窗/抽屉式）
- OCR 审核页
- 趋势页
- 单次检查详情页

## 3. 页面定义

### 3.0 首页空状态（首次使用）

**实现状态**: Verified (v2.0.0) | **路径**: `src/app/page.tsx`


触发条件：系统中无任何成员档案

展示内容：
- 欢迎文案："欢迎使用家庭检查单管理"
- 引导说明："记录家人健康足迹，从添加第一位成员开始。我们将为您智能分析等核心指标趋势。"
- 操作按钮：「添加第一位成员」（主按钮）
- 无顶部导航，无底部操作栏

交互逻辑：
- 点击「添加第一位成员」→ 跳转成员档案创建页
- 创建成功后自动返回首页，展示成员卡片

### 3.1 首页（有成员后）

**实现状态**: Updated (v2.1.0) | **路径**: `src/app/page.tsx`


展示内容：
- 顶部标题："家庭成员" + 右侧设置入口
- 成员卡片列表（支持上下滑动）
  - 成员头像/首字母
  - 成员名称 + 成员类型标签（儿童/成人/老人）
  - 最近一次检查时间（来自 API `last_check_date`，无数据时显示"尚无记录"）
  - 待审核检查单数（来自 API `pending_review_count`，大于 0 时显示红色标签）
- 底部固定导航栏：成员 / 审核箱
- 「+ 添加成员」按钮（位于成员卡片列表底部，虚线边框）

交互逻辑：
- 点击成员卡片 → 进入该成员的 Dashboard 页
- 点击「录入新检查单」→ 弹出上传页
- 点击「+ 添加成员」→ 跳转成员档案创建页
- 点击底部「审核箱」→ 进入 OCR 审核页

### 3.2 成员 Dashboard 页

**实现状态**: Updated (v2.3.0) | **路径**: `src/app/members/[id]/page.tsx`


展示内容：
- 顶部导航：返回箭头 + 成员名称 + 编辑按钮
- 最后更新时间戳
- 指标卡片区域（根据 member_type 动态展示）
  - **儿童（child）**：
    - 近视防控看板：眼轴趋势卡片 + 视力趋势卡片 + EyeModeToggle（双眼/左眼/右眼切换）+ 最近两次检查对比
    - 生长发育看板：身高趋势图 + 体重趋势图 + 最近两次检查对比
  - **老人（senior）**：
    - 指标详情：动态展示已有数据的指标卡片（身高、体重、眼轴、视力等），无数据不展示
    - 空状态：显示"暂无指标数据，请上传检查单"
  - **成人（adult）**：同老人逻辑
- 待审核项卡片（如有 pending_review 状态的检查单）
  - 显示检查日期和类型
  - 「去审核」按钮
- 底部固定按钮：「录入新检查单」

交互逻辑：
- 点击指标卡片 → 进入趋势页
- 点击「去审核」→ 进入 OCR 审核页
- 点击「录入新检查单」→ 弹出上传页

### 3.3 成员档案创建/编辑页

**实现状态**: Verified (v2.0.0) | **路径**: `src/app/members/new/page.tsx`, `src/app/members/[id]/edit/page.tsx`


展示内容：
- 表单字段：
  - 姓名（必填，文本输入）
  - 性别（必填，单选：男/女）
  - 出生日期（必填，日期选择器）
  - 成员类型（必填，单选：儿童 0-18 岁 / 成人 18-60 岁 / 老人 60 岁以上）
- 操作按钮：
  - 创建页：「保存并开始记录」（主按钮）
  - 编辑页：「更新信息」（主按钮）+「删除此成员」（危险按钮，二次确认）

交互逻辑：
- 创建页：保存后返回首页，自动展示新成员卡片
- 编辑页：保存后返回 Dashboard，数据实时更新
- 删除成员：弹出二次确认「确认删除？历史检查数据将保留但不再展示」→ 确认后执行软删除，返回首页

### 3.4 上传页（弹窗/抽屉式）

**实现状态**: Verified (v2.0.0) | **路径**: `src/components/upload/UploadOverlay.tsx`


展示内容：
- 上传区域（点击触发文件选择器）
- 文件类型说明：支持 JPG、PNG、PDF
- 处理进度条（上传中 → 脱敏中 → 识别中）
- 取消按钮

交互逻辑：
- 选择文件后自动开始上传
- 上传过程中显示进度条
- 上传成功：
  - 识别成功 → 自动关闭弹窗，Dashboard 图表刷新
  - 识别失败 → 提示"智能识别失败"，提供「重新上传」和「手工录入」选项
- 点击取消 → 关闭弹窗，已上传的文件继续后台处理

### 3.5 OCR 审核页

**实现状态**: Verified (v1.5.0) | **路径**: `src/review/page.tsx`


展示内容：
- 顶部导航：返回箭头 + "OCR 识别结果审核"
- 左右分栏（桌面端）/ 上下分栏（移动端）：
  - 左侧/上方：脱敏图预览（隐藏姓名、条码等敏感区域）
  - 右侧/下方：结构化字段表单
    - 检查日期（可编辑）
    - 各指标字段（可编辑，冲突字段标红）
    - 单位显示在字段右侧
- 置信度提示条：
  - ≥ 0.8：绿色，"识别结果可信"
  - 0.6-0.8：黄色，"请核对关键字段"
  - < 0.6：红色，"识别置信度低，请仔细核对"
- 规则冲突提示区（如有）：
  - 红色警告卡片，列出具体冲突原因
  - 例："血糖单位不匹配：识别为 mg/dL，应为 mmol/L"
  - 例："眼轴数据缺失单侧记录"
- 底部操作按钮：
  - 「退回重识别」（次要按钮）
  - 「保存草稿」（次要按钮）
  - 「确认入库」（主按钮）

交互逻辑：
- 用户可修改任意字段值
- 修改后字段旁显示"已修改"标记
- 点击「确认入库」→ 写入正式指标表 → 返回 Dashboard
- 点击「退回重识别」→ 标记为 review_rejected → 可重新触发 OCR
- 点击「保存草稿」→ 暂存当前修改 → 返回列表，状态保持 pending_review

### 3.6 趋势页

**实现状态**: Updated (v2.3.0) | **路径**: `src/app/members/[id]/trends/page.tsx`


展示内容：
- 顶部导航：返回箭头 + "趋势分析"
- 指标切换标签栏：眼轴 / 身高 / 体重 / 视力 / 血糖 / 血脂 / **血红蛋白** / **糖化血红蛋白**
- 时间范围切换：1 月 / 3 月 / 6 月 / 1 年
- EyeModeToggle 切换器（仅眼轴/视力显示）：双眼 / 左眼 / 右眼
- 图表区域：
  - 折线图（左右眼并列展示时区分实线/虚线，单眼模式时另一眼淡化）
  - 绿色参考区间背景带
  - 超阈值数据点标红
- 指标概览侧边栏（右侧）：
  - 参考区间
  - 最新状态
  - 预计年增长（仅身高显示）
- 历史记录列表：
  - 按日期分组展示。
  - 同一天的左右眼数据并列展示，不求平均值。
  - 显示日期、检查侧（左/右/全）、数值。
- 报警说明卡片（如有异常）：
  - 红色警告图标
  - 异常说明文字（如"增长速度略高于同龄参考"）

交互逻辑：
- 切换指标 → 图表和数据同步刷新
- 切换时间范围 → 图表 X 轴范围调整
- 点击数据点 → 弹出该次检查的详情卡片

### 3.7 单次检查详情页

**实现状态**: Verified (v2.0.0) | **路径**: `src/app/members/[id]/records/[recordId]/page.tsx`


展示内容：
- 检查日期和机构名称
- 脱敏图预览（可放大）
- 所有指标列表（指标名、数值、单位、是否异常标记）

交互逻辑：
- 从趋势页点击数据点进入
- 返回按钮回到趋势页

## 4. 图表策略

### 4.1 生长发育

- 身高折线图
- 体重折线图
- 标准百分位参考带
- 年增长速度摘要

### 4.2 视力与眼轴

- 眼轴趋势图
- 视力或屈光度趋势图
- 左右眼并列展示
- 参考区间与异常提示（基于成员年龄动态适配）

### 4.3 血糖与血脂

- 折线图叠加正常区间
- 超阈值数据点高亮红色
- 指标越界说明卡片

### 4.4 血红蛋白

- 折线图叠加正常区间（男性/女性参考区间不同）
- 超阈值数据点高亮红色

## 5. 状态设计

- 空状态：无检查单、无趋势数据、无待审核项
- 处理中：上传中、脱敏中、OCR 中、校验中
- 异常状态：识别失败、规则冲突、待人工审核
- 成功状态：审核完成并入库

## 6. 前端实现要求

- 前端可先基于 Mock Data 开发
- 页面组件必须与 API Contract 解耦
- 图表配置集中管理，避免散落在页面代码中
