# 家庭检查单管理应用 PRD

## 1. 项目目标

构建一个面向家庭场景的健康检查单管理应用，支持儿童检查单上传、OCR 结构化、人工审核、趋势图查看和基础风险提示。

第一阶段聚焦以下指标：
- 身高
- 体重
- 血糖
- 血脂
- 低密度脂蛋白
- 眼轴
- 视力

## 2. 核心用户

- 家长或监护人
- 需要长期观察儿童视力与生长发育的家庭成员

## 3. 核心问题

- 检查单分散在相册、微信和纸质文件中，难以长期追踪
- 家长难以快速判断 3 个月内指标变化趋势
- 医疗表格 OCR 易错，普通图片管理工具不具备纠错与审核链路

## 4. MVP 范围

### 4.1 必做能力

- 儿童档案管理
- 检查单上传与原件保存
- 本地脱敏后发送 OCR
- 多模态 OCR 候选结构化抽取
- 规则引擎校验
- 人工确认与修订
- 结构化入库
- 3 个月趋势图与报警提示

### 4.2 暂不纳入

- 医嘱自动生成
- 多医院标准模板自动识别优化
- 家庭成员复杂权限体系
- 原生 App

## 5. 关键业务流程

1. 家长创建儿童档案
2. 上传检查单图片或 PDF
3. 系统在本地进行敏感区域遮罩或裁切
4. 将脱敏图发送给公有云多模态模型做候选抽取
5. 本地规则引擎进行单位、阈值、左右眼与字段完整性校验
6. 若异常则进入人工审核
7. 审核通过后入正式指标表
8. 用户在趋势页查看 3 个月趋势、参考区间和报警提示

## 6. 成功标准

- 用户可以在 3 分钟内完成一次检查单上传与确认
- OCR 高置信样本能够稳定完成结构化候选抽取
- 错误单位或明显串位数据不能进入正式指标表
- 家长可以直接看到视力、眼轴、生长发育、血糖血脂趋势

## 7. 关键约束

- 不向公有云发送包含姓名、条码、就诊号等敏感区域的原图
- 大模型输出默认不可信，必须经过规则校验和人工确认
- 所有图表必须优先表达阈值、区间与变化率，不只显示折线
