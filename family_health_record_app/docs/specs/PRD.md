# 家庭检查单管理应用 PRD

## 1. 项目目标

构建一个面向家庭场景的健康检查单管理应用，支持家庭成员（侧重儿童发育与成人慢病）检查单上传、OCR 结构化、人工审核、趋势图查看和基础风险提示。

第一阶段聚焦以下指标：
- 身高
- 体重
- 血糖
- 总胆固醇（TC）
- 甘油三酯（TG）
- 高密度脂蛋白（HDL）
- 低密度脂蛋白（LDL）
- 眼轴
- 视力

## 2. 核心用户

- 家长或监护人
- 需要长期观察儿童视力与生长发育，或管理成员血糖血脂的家庭核心成员

## 2.1 成员类型与适用范围

MVP 支持以下成员类型：
- `child`：儿童或青少年，重点关注身高、体重、视力、眼轴
- `adult`：成年人，重点关注血糖、血脂
- `senior`：老人，第一阶段与 `adult` 使用同一套代谢类规则

MVP 中的产品与算法差异：
- 儿童默认首页展示生长发育与视力模块
- 成人和老人默认首页展示血糖与血脂模块
- 儿童启用 `baseline_age_months`、百分位参考带和增长速度计算
- 成人和老人不展示儿童生长百分位，也不启用远视储备算法

## 3. 核心问题

- 检查单分散在相册、微信和纸质文件中，难以长期追踪
- 家长难以快速判断 3 个月内指标变化趋势
- 医疗表格 OCR 易错，普通图片管理工具不具备纠错与审核链路

## 4. MVP 范围

### 4.1 必做能力

- 家庭成员档案管理（支持多成员切换）
- 检查单上传与原件保存
- 服务端私有环境完成脱敏后发送 OCR
- 多模态 OCR 候选结构化抽取
- 规则引擎校验
- 人工确认与修订
- 结构化入库
- 3 个月趋势图与报警提示

### 4.2 暂不纳入

- 医嘱自动生成
- 多医院标准模板自动识别优化
- 家庭成员复杂权限体系
- 原生 App

## 4.3 MVP 权限边界（内网免登录）

- **本应用部署于内网环境，无需登录认证**。用户打开浏览器访问内网地址即可直接使用。
- MVP 默认单实例部署，所有数据归属于同一家庭空间。
- 不支持多租户、跨账户共享、成员级细粒度授权或审计审批流。
- 成员删除默认采用软删除，不直接物理清除历史检查记录。
- 任何能访问该内网地址的用户均拥有该家庭空间内全部成员的读写权限。

## 5. 关键业务流程

### 5.0 首次使用引导（空状态 → 创建成员）

1. 用户首次打开应用，进入首页
2. 系统检测到无成员档案，展示空状态引导页
3. 用户点击「添加第一位成员」
4. 填写成员档案（姓名、性别、出生日期、成员类型）
5. 保存后返回首页，展示成员卡片
6. 用户可继续添加更多成员，或开始录入检查单

### 5.1 核心业务链路（上传 → 审核 → 趋势）

1. 用户在首页选择或切换成员
2. 上传检查单图片或 PDF
3. 系统在私有部署的后端环境中进行敏感区域遮罩或裁切
4. 将脱敏图发送给公有云多模态模型做候选抽取
5. 本地规则引擎进行单位、阈值、左右眼与字段完整性校验
6. 候选结果先写入原始抽取结果表
7. 若异常则进入人工审核
8. 审核通过后入正式指标表
9. 用户在趋势页查看 3 个月趋势、参考区间和报警提示

## 5.2 审核触发条件

以下情况必须进入人工审核：
- OCR 置信度低于系统阈值
- 单位与规则库不匹配
- 左右眼字段缺失或疑似串位
- 同一张检查单中同一指标出现重复冲突值
- 检查日期缺失或无法解析
- OCR 接口超时或结构化结果不完整

## 5.3 状态流转

标准流转：

`uploaded -> desensitizing -> ocr_processing -> rule_checking -> pending_review -> approved -> persisted`

异常流转：

`ocr_failed`

`rule_conflict`

`review_rejected`

## 5.4 数据留存与追溯

- 原始文件、脱敏文件、原始 OCR JSON、人工修订记录和正式 observation 分层存储
- 原始 OCR JSON 不直接参与趋势计算
- 人工审核必须保留修改痕迹与时间戳
- 正式 observation 是图表与分析接口的唯一数据来源

## 6. 成功标准

- 标准样本上传后，90% 以上可在 3 分钟内完成上传与确认
- OCR Golden Set 上关键字段候选抽取准确率达到可配置门槛，低置信结果必须进入人工审核
- 错误单位、左右眼串位、重复冲突值不能进入正式指标表
- 用户可以直接看到视力、眼轴、生长发育（儿童）以及血糖血脂（全员）的趋势图表

## 6.1 报警与图表规则

- 儿童身高和体重：显示年龄参考百分位或参考带，并显示增长速度摘要
- 儿童眼轴和视力：显示左右眼趋势、年龄参考区间和异常提示
- 成人与老人血糖血脂：显示正常区间、超阈值红区和越界说明
- 报警仅作为健康管理提示，不作为诊断结论

## 7. 关键约束

- 不向公有云发送包含姓名、条码、就诊号等敏感区域的原图
- 默认由后端私有服务完成脱敏，浏览器端不直接把原图发送给公有云 OCR
- 大模型输出默认不可信，必须经过规则校验和人工确认
- 所有图表必须优先表达阈值、区间与变化率，不只显示折线
