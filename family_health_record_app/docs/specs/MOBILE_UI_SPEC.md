# 移动端 UI 规格说明

> **版本**: v1.0.0  
> **基于**: Web 版 UI_SPEC.md (v2.3.0)  
> **最后更新**: 2026-04-04

## 1. 设计原则

复用 Web 版设计原则，针对移动端做以下适配：

- **移动端优先**：首屏展示核心信息，复杂功能下沉
- **手势交互**：滑动返回、下拉刷新、左右滑动切换指标
- **底部导航**：替代 Web 版的顶部导航栏
- **单列布局**：放弃 Web 版的左右分栏，采用上下堆叠
- **触控友好**：按钮最小 44px 点击区域

## 2. 信息架构

与 Web 版一致，共 8 个页面：

| 页面 | 路由 | 说明 |
|------|------|------|
| 首页空状态 | `/` | 首次使用引导 |
| 首页成员列表 | `/` | 有成员后展示 |
| 成员 Dashboard | `/member/:id` | 指标卡片展示 |
| 成员创建/编辑 | `/member/new`, `/member/:id/edit` | 表单页 |
| 上传页 | BottomSheet | 底部抽屉式 |
| OCR 审核页 | `/review/:taskId` | 审核工作台 |
| 趋势页 | `/member/:id/trends` | 图表展示 |
| 检查详情页 | `/member/:id/record/:recordId` | 单次记录 |

## 3. 页面定义（移动端适配）

## 6. 状态设计

与 Web 版一致，新增移动端特有状态：

| 状态 | 说明 |
|------|------|
| `pull_to_refresh` | 下拉刷新中 |
| `bottom_sheet_open` | 底部抽屉展开中 |
| `image_zoomed` | 图片已放大 |
| `tab_switching` | 标签切换中 |

## 7. 响应式断点

| 断点 | 宽度 | 适配 |
|------|------|------|
| `sm` | < 640px | 手机竖屏 |
| `md` | 640-1024px | 手机横屏/小平板 |
| `lg` | > 1024px | 平板/折叠屏 |

移动端实现主要针对 `sm` 断点。

## 8. 与 Web 版差异对照

| 项目 | Web 版 | 移动版 |
|------|--------|--------|
| 布局 | 左右分栏 | 上下堆叠 |
| 导航 | 顶部 + 左侧 | 底部 Tab |
| 上传 | 弹窗 | BottomSheet |
| 表单 | 横向排列 | 纵向堆叠 |
| 图表 | 侧边栏 | 图表下方 |
| 触控 | 无 | 手势支持 |
