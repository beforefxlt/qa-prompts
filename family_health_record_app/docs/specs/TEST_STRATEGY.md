# 家庭检查单管理应用测试策略

> **版本**: v1.2.0
> **最后更新**: 2026-04-02
> **变更说明**: 新增基础设施健康检查、OCR 提示词管理测试、测试代码检查门禁

## 1. 测试目标

确保应用在家庭场景下可安全、稳定地完成检查单采集、OCR 结构化、人工审核与趋势展示，并避免脏数据、隐私泄漏和错误图表结论。

## 2. 分层测试

### 2.1 单元测试

覆盖：
- 单位校验（mmol/L vs mg/dL 拦截）
- 年龄换算（date_of_birth → baseline_age_months）
- 派生指标计算（眼轴增长率、身高增长速度）
- 趋势聚合（3 个月窗口数据切片）
- 阈值判断（越界标记 is_abnormal）
- 软删除过滤（is_deleted=false 查询）

### 2.2 基础设施健康检查

覆盖：
- MinIO bucket 存在性与可访问性
- MinIO 上传下载功能验证
- MinIO 目录结构完整性

### 2.3 OCR Golden Set 回归

覆盖：
- 文档分类（视力检查单 vs 生长发育 vs 代谢类）
- 表格抽取准确率
- 左右眼区分
- 单位提取与标准化
- 不同年龄/性别样本的参考值规则匹配
- 错误样本阻断

### 2.4 API 合约测试

覆盖：
- 请求响应 Schema（所有资源对象）
- 状态码（200/201/400/404/409/500）
- 成员档案 CRUD（创建、查询、更新、软删除）
- 审核流状态迁移（uploaded → persisted 全链路）
- 非法输入拦截（空姓名、非法日期、错误枚举值）
- 幂等性（重复提交 OCR 不产生重复 observation）

### 2.5 集成测试

覆盖：
- 上传 → 脱敏 → OCR → 规则校验 → 人工审核 → 入库 → 趋势查询（全链路）
- 规则冲突 → 生成 review_task → 人工修正 → 入库
- OCR 置信度低 → 进入审核页 → 人工确认 → 入库
- 软删除成员后，其检查单数据保留但不在前端展示

### 2.6 前端 E2E

覆盖：
- 首次使用：空状态引导 → 创建成员 → 返回首页展示卡片
- 上传流程：选择图片 → 进度展示 → 识别成功 → 图表刷新
- OCR 审核流程：进入审核页 → 修改冲突字段 → 确认入库
- 趋势图切换：切换指标/时间范围 → 图表与数据同步刷新
- 报警红区展示：超阈值数据点标红、参考区间绿色带
- 空状态和错误态：无成员、无检查单、识别失败、网络断开
- 成员管理：创建、编辑、软删除成员档案

### 2.7 非功能测试

覆盖：
- 隐私脱敏（页头页脚敏感区域遮罩验证）
- OCR 接口超时（>30s 自动降级为 ocr_failed）
- MinIO 不可用（上传失败 → 友好提示）
- 磁盘空间不足（脱敏文件写入失败 → 清理临时文件）
- 重复提交与幂等（同一 document_id 重复 submit-ocr）

### 2.8 测试代码检查

覆盖：
- 生产代码中无 mock/stub/fixture 逻辑
- 无硬编码测试数据
- 无 `if "test" in filename` 类判断
- 自动化检查：`python scripts/check_no_test_code.py`

## 3. P1-P5 风险视角

- `P1`：标准上传、审核、查询主路径；首次使用空状态引导；成员档案 CRUD
- `P2`：非法单位、重复值、缺字段、错误文件格式；成员信息编辑校验
- `P3`：MinIO 不可用、OCR 接口超时、磁盘权限异常、网络断开时的草稿保存
- `P4`：年龄月龄边界（儿童 vs 成人 vs 老人参考值档位切换）、3 个月窗口切换边界、指标异常阈值上下界
- `P5`：空状态引导文案是否清晰、OCR 审核页冲突提示是否可理解、图表是否能直接读懂、错误提示是否友好

## 4. 验收门禁

- 单元测试通过
- 基础设施健康检查通过
- OCR Golden Set 回归通过
- API 合约测试通过
- 关键 E2E 主链路通过（含首次使用引导）
- 隐私样本检查通过
- 测试代码检查通过 (`scripts/check_no_test_code.py`)
- `verify-requirements` 核验通过
