# 家庭检查单管理应用测试策略

## 1. 测试目标

确保应用在家庭场景下可安全、稳定地完成检查单采集、OCR 结构化、人工审核与趋势展示，并避免脏数据、隐私泄漏和错误图表结论。

## 2. 分层测试

### 2.1 单元测试

覆盖：
- 单位校验
- 年龄换算
- 派生指标计算
- 趋势聚合
- 阈值判断

### 2.2 OCR Golden Set 回归

覆盖：
- 文档分类
- 表格抽取
- 左右眼区分
- 单位提取
- 错误样本阻断

### 2.3 API 合约测试

覆盖：
- 请求响应 Schema
- 状态码
- 审核流状态迁移
- 非法输入拦截

### 2.4 集成测试

覆盖：
- 上传
- 脱敏
- OCR
- 规则校验
- 人工审核
- 入库
- 趋势查询

### 2.5 前端 E2E

覆盖：
- 上传流程
- OCR 审核流程
- 趋势图切换
- 报警红区展示
- 空状态和错误态

### 2.6 非功能测试

覆盖：
- 隐私脱敏
- 队列失败重试
- 对象存储不可用
- 大模型超时
- 重复提交与幂等

## 3. P1-P5 风险视角

- `P1`：标准上传、审核、查询主路径
- `P2`：非法单位、重复值、缺字段、错误文件格式
- `P3`：MinIO 不可用、Redis 不可用、OCR 接口超时、磁盘权限异常
- `P4`：年龄月龄边界、3 个月窗口切换边界、阈值上下界
- `P5`：家长首次使用、错误提示是否可理解、图表是否能直接读懂

## 4. 验收门禁

- 单元测试通过
- OCR Golden Set 回归通过
- API 合约测试通过
- 关键 E2E 主链路通过
- 隐私样本检查通过
- `verify-requirements` 核验通过
