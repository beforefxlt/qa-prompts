# BUG-UI-001: 测试用例与 UI 文案不一致

## 问题描述
E2E 测试用例硬编码的文案与实际页面不符，导致测试失败。

## 根因分析
1. 测试用例凭记忆编写文案，未对照 UI_SPEC.md 或代码
2. 缺少文案常量层，测试和代码各自硬编码

## 影响范围
- E2E 测试 7 项失败
- 测试维护成本高

## 文档 vs 代码差异

| 位置 | 文档定义 | 代码实现 | 决策 |
|------|---------|---------|------|
| 首页引导说明 | "请先添加家庭成员，才能开始记录检查数据" | "记录家人健康足迹，从添加第一位成员开始。我们将为您智能分析等核心指标趋势。" | 保留代码，更新文档 |
| 创建页按钮 | "保存并继续" | "保存并开始记录" | 保留代码，更新文档 |
| 编辑页删除按钮 | "删除成员" | "删除此成员" | 保留代码，更新文档 |

## 修复方案
1. ✅ 创建 `src/constants/ui-text.ts` 文案常量层
2. ✅ 更新测试用例引用常量
3. ✅ 更新 UI_SPEC.md 与代码一致
4. ⏳ 添加 CI 门禁检查文案一致性
