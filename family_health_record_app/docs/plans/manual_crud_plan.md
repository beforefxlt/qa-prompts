# Implementation Plan - 手动录入与指标生命周期管理 (CRUD)

> **目标**: 使成员可以脱离图片 OCR 的限制，直接手动录入健康数据，并对所有历史记录进行增删改查。

---

## 1. 需求分析 (Requirement Analysis)

1.  **手动录入**: 支持跳过 OCR 图片，直接通过表单输入检查日期、机构及各项指标。
2.  **管理功能 (CRUD)**: 允许用户对已录入（无论是 OCR 还是手动）的数据进行管理（修改或删除）。
3.  **数据一致性**: 手动更新后，直接覆盖旧值，不需要保留原始 OCR 结果或追溯历史。
4.  **校验逻辑**: 在后端 Pydantic Schema 中引入常规合理性区间校验（Sanity Check），防止手滑误录（如身高 1000cm）。

---

## 2. 方案设计 (Detailed Design)

### 2.1 数据库调整 (Database Schema)
-   **`ExamRecord` 表**:
    -   `document_id`: 修改为 `nullable=True` (允许为 `null`)。
-   **`Observation` 表**:
    -   现有结构已支持，无需变动。

### 2.2 接口契约更新 (API Contract)
| 类型 | 路径 | 说明 | 校验逻辑 |
|:---|:---|:---|:---|
| `POST` | `/api/v1/members/{id}/manual-exams` | 手动提交一次检查数据 | 指标数值在常规合理区间内 |
| `PATCH` | `/api/v1/observations/{id}` | 修改单条指标值 | 数值范围校验 |
| `DELETE` | `/api/v1/exam-records/{id}` | 级联删除某次检查及其所有指标 | - |

### 2.3 常规合理性校验 (Sanity Check Ranges)
| 指标 | 常规区间 (Min - Max) | 单位 |
|:---|:---|:---|
| `axial_length` | 15.0 - 35.0 | mm |
| `height` | 30.0 - 250.0 | cm |
| `weight` | 2.0 - 500.0 | kg |

---

## 3. UI 交互逻辑 (UX Design)

1.  **入口**: 在成员看板 (Dashboard) 的“录入新检查单”按钮中，增加二级菜单：“📸 拍照记录” / “✍️ 手动录入”。
2.  **手动录入弹窗**:
    -   选择日期（默认今天）。
    -   录入机构。
    -   指标动态列表：预设该成员常用指标（如儿童默认显示身高/体重/眼轴），支持删除或添加更多。
3.  **管理操作**:
    -   在趋势分析页面的“历史记录”列表中，为每行增加操作菜单（...）。
    -   包含“编辑”和“删除”功能。

## 5. 验收标准 (Definition of Done)
1.  成功手动录入一次身高数据，且能在趋势图展示。
2.  输入超出范围的值（如眼轴 100mm）时，后端返回 422 错误。
3.  删除一次历史检查记录后，该记录从数据库、列表、趋势图中同步消失。
4.  修改一条 OCR 数据后，界面显示最新值。
