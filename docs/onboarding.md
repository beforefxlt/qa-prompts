# 测试团队 AI 辅助入门指南

> 从零开始，用 Prompt 模板库高效生成测试用例

**版本**: v1.0
**适用对象**: 首次使用 AI 辅助测试的团队成员

---

## 一、你需要准备什么

### AI 工具选择

| 工具 | 推荐度 | 说明 |
| :--- | :--- | :--- |
| ChatGPT (GPT-4) | 推荐 | Markdown 表格输出遵从度高，Mermaid 支持好 |
| Claude | 推荐 | 长文本理解和结构化输出能力强 |
| 通义千问 | 可用 | 中文理解好，Mermaid 语法支持待验证 |
| Kimi | 可用 | 长上下文支持好，适合大段需求分析 |

> 团队统一使用的工具请以内部通知为准。如遇到特定模型对某模板输出质量不佳的情况，请在仓库 Issues 中反馈（标签：`prompt-quality`）。

### 获取模板

```bash
# 克隆仓库
git clone <仓库地址>

# 进入模板目录
cd qa-prompts/templates/
```

## 二、选择模板：我该用哪个？

根据你当前的工作阶段和任务类型，使用以下决策树快速定位：

```
你现在要做什么？
│
├── 评审需求 / 审查 PRD
│   └── → REQ-001 需求可测试性与歧义审查
│
├── 设计测试用例
│   │
│   ├── 输入框 / 表单字段 / API 参数的边界测试？
│   │   └── → BVA-001 边界值分析生成器
│   │
│   ├── 订单 / 工作流 / 审批等状态流转的测试？
│   │   └── → STM-001 状态机流转生成器
│   │
│   └── RESTful 接口的契约和逻辑测试？
│       └── → API-001 API 契约与业务逻辑测试
│
└── 探索性测试 / 自由测试
    └── → EXP-001 五边形人格探索性测试
```

## 三、端到端工作流

### 第一步：填写模板变量

1. 打开模板文件（如 `templates/bva-boundary-value-analysis.md`）
2. 找到 `## 模板定义` 中的代码块
3. 将代码块内容复制到 AI 对话框
4. 替换所有 `{{变量名}}` 为实际值

**示例**：
```
# 替换前
当前正在进行 [项目名称：{{project_name}}] 的系统测试。

# 替换后
当前正在进行 [项目名称：退休返聘系统] 的系统测试。
```

### 第二步：发送给 AI 并获取输出

- 将填写完毕的 Prompt 发送给 AI 工具
- 等待 AI 生成测试用例

### 第三步：质检 AI 输出

**必做检查项**（适用于所有模板）：

- [ ] **完整性**：AI 是否覆盖了模板 Constraints 中要求的所有维度？
- [ ] **准确性**：预期结果是否符合业务逻辑？（AI 可能"编造"不存在的规则）
- [ ] **可执行性**：每条用例是否有明确的输入、操作步骤和预期结果？
- [ ] **无重复**：是否有语义重复的用例？
- [ ] **优先级合理**：P0/P1/P2 的分配是否与业务重要性匹配？

### 第四步：录入和管理

将通过质检的用例录入你们的测试管理工具（如 TestRail、禅道、飞书多维表格等）。

**建议**：
- 保留 AI 原始输出作为备份（方便追溯模板效果）
- 在测试管理工具中标注"AI 辅助生成"，便于后续统计采纳率

## 四、常见问题

### Q1：AI 生成的用例漏掉了关键场景怎么办？

检查你填写的变量是否足够详细。常见原因：
- `{{constraints}}` 写得太笼统（如"有限制"），应改为具体数值（如"18-65"）
- `{{business_logic}}` 遗漏了重要的业务规则

如果变量填写已经很详细但仍然遗漏，请提交 Issue 反馈，我们会优化模板约束。

### Q2：AI 输出的格式不对（比如不是表格）怎么办？

- 在 Prompt 末尾追加一句："请严格按照上述 Output Format 中的表格格式输出"
- 换一个模型试试（不同模型对格式指令的遵从度不同）

### Q3：Mermaid 图渲染不出来？

部分 AI 工具不支持 Mermaid 预览。可以：
1. 复制 Mermaid 代码
2. 粘贴到 [Mermaid Live Editor](https://mermaid.live/) 在线渲染
3. 或在支持 Mermaid 的 Markdown 编辑器中查看（如 VS Code + Mermaid 插件）

### Q4：多个模板可以组合使用吗？

可以。推荐组合：
- **需求阶段**：先用 REQ-001 审查需求，再根据审查结果选择 BVA/STM/API 生成用例
- **功能模块**：先用 STM-001 梳理状态流转，再用 BVA-001 对关键字段做边界测试
- **接口测试**：API-001 覆盖契约和逻辑后，用 EXP-001 做补充探索

### Q5：我想贡献新模板或改进现有模板？

参考 `docs/engineering-guide.md` 中的模板开发 Checklist，提交 PR 即可。
